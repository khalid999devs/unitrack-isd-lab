## Team Communication Rules

1. Each member should update their Jira task status honestly.
2. Each member should inform the team if they face any blocker.
3. Each member should use the Git workflow properly.
4. Each member should include Jira task ID in branch names and commit messages.
5. The team should discuss before changing major project structure.
6. No one should directly push to the `main` branch.

## Task Ownership Rule

Each task should have one main owner.

For example:

```text
Login UI -> Sadik
Login backend -> Siyam
Login integration -> Khalid + Sadik
Jira workflow -> Khalid
Database schema -> Siyam
```

## Collaboration Rule

Even if one member owns a task, other members can help. But the Jira assignee should be the main responsible person for completing that task.

## Review Rule

Before a task is moved to Done:

1. The assigned member completes the work
2. The code is pushed to GitHub
3. Pull request is created if needed
4. Another member or Scrum Master checks it
5. The task is tested
6. Then it is moved to Done
