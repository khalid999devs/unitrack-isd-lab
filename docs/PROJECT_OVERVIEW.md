# Project Overview - UniTrack ISD Lab

## Project Name

UniTrack: Student Academic Resource Management System

## Short Description

UniTrack is a simple web-based academic management system where students, teachers, and admins can manage basic academic activities in one platform. It helps organize courses, class routines, notices, study materials, and assignments in a structured way.

## Project Purpose

The purpose of this project is to develop a small but complete academic management system while properly following software development practices, Scrum workflow, Jira task management, and GitHub collaboration.

The main focus is not only the final software, but also the development process, team collaboration, task tracking, branching, commits, pull requests, and project documentation.

## User Roles

The system has three main user roles:

1. Student
2. Teacher
3. Admin

## Role-wise Responsibilities

### Student

Students can access their academic dashboard and view academic information.

Main actions:

1. Login to the system
2. Manage student profile
3. View courses
4. View class routine
5. View notices
6. View or download study materials
7. View assignments or tasks
8. Search and filter academic resources

### Teacher

Teachers can manage academic content related to courses, notices, study materials, and assignments.

Main actions:

1. Login to the system
2. Manage teacher profile
3. View assigned courses
4. Upload study materials
5. Create notices
6. Create assignments or tasks
7. Manage academic content

### Admin

Admins can manage the main system data and control academic information.

Main actions:

1. Login to the system
2. Manage students
3. Manage teachers
4. Manage courses
5. Manage class routines
6. Manage notices
7. Manage dashboards and system data

## Project Scope

This project will be developed as a medium-small academic management system. The goal is to keep the system simple, understandable, and suitable for an ISD Lab project.

The first version will focus on:

1. Authentication and role-based access
2. Basic profile management
3. Course and routine management
4. Notice management
5. Study material upload and view
6. Assignment or task management
7. Basic dashboards
8. Search and filter

## Out of Scope for First Version

To keep the project simple, the following advanced features are not included in the first version:

1. AI-based summary generation
2. Payment system
3. Real-time chat
4. Mobile app
5. Advanced analytics
6. Template marketplace
7. Complex notification system

## Development Methodology

The team will follow Agile Scrum.

Main Scrum activities:

1. Product backlog creation
2. Sprint planning
3. Sprint backlog creation
4. Task assignment
5. Daily progress update
6. Sprint review
7. Sprint retrospective

## Sprint Plan Summary

### Sprint 1: Project Setup and Planning

Goal: Set up project foundation, Jira workflow, GitHub branch workflow, and initial requirement documentation.

### Sprint 2: Core Module Development

Goal: Develop authentication, profiles, course management, and routine management.

### Sprint 3: Academic Content and Finalization

Goal: Develop notices, study material upload, assignments, dashboards, search/filter, testing, and documentation.

## Success Criteria

The project will be considered successful if:

1. Main features are implemented according to scope
2. Jira board shows proper task tracking
3. GitHub repository shows proper branch and commit workflow
4. Team members work with assigned tasks
5. Sprint progress is documented
6. Final presentation includes system design and Scrum process
